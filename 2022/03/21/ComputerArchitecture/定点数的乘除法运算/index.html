补码一位乘法
Booth法

假定被乘数X和乘数Y均为用补码表示的纯小数，分别为：

$[X]_补 = X_{0}.X_{-1}X_{-2}X_{-3}···X_{-(n-1)}$

$[Y]_补 = Y_{0}.Y_{-1}Y_{-2}Y_{-3}···Y_{-(n-1)}$

$[X*Y]_补 = [X]_补[(Y_{-1}-Y_{0})2^{0}+(Y_{-2}-Y_{-1})2^{-1}+···+(Y_{-(n-1)}-Y_{-(n-2)})2^{-(n-2)}+(0-Y_{-(n-1)})2^{-(n-1)}]$

由此得到运算过程中的4种操作（背就完事了！）：

Y_i Y_(i-1) Y_(i-1)-Y_i 操作
0 0 0 +0，右移1次
0 1 1 +[X]补，右移1次
1 0 1 +[-X]补，右移1次
1 1 0 +0，右移1次

根据以上，Booth法的运算法则描述如下：

 1. 乘数与被乘数均用补码表示，连符号位一起参加运算
 2. 乘数最低位后增加一个附加位(可以用A_-1表示），初始预设为0
 3. 从附加位开始，依据上表的操作规律完成运算

除法运算

原码除法运算

原码除法的法则应包括：

①除数≠0；定点纯小数时，｜被除数｜＜｜除数｜；定点纯整数时，｜被除数｜ ＞｜除数｜。

②与原码乘法类似的是原码䲔法商的符号和商的值也是分别处理的，商的符号等于被除数的符号与除数的符号相异或。

③商的值就等于被除数的绝对值除以除数的绝对值。

④最后将商的符号与商的值拼接在一起就得到原码除法的商。

恢复余数法

对于定点纯小数：

①其过程是符号位单独处理：商符号为被除数和除数符号异或的结果，余数的符号和被除数相同

②余数（被除数）左移一位，左移补上的末位数字与商位相同，减除数。

③若够减，上商为 1；若不够减，上商为0，同时加除数——恢复余数。

重复上述的步骤②③直到除尽或精度达到要求。最后， 拼接商符得到商，即可获得除法的结果。余数的尾数为余数（被除数）列中在不断左移的过程中仍保留的数（不理解可以看下面的过程，加粗字为仍保留的数，细体为左移新生成的数），余数的阶数为-1*[左移次数]

注意：如果最后的商位为0，需要给得到的余数再+|Y|恢复余数得到正确的余数。

若被除数数 X＝－0.10001011，除数 Y＝0.1110，利用原码求商及余数。

该例满足|X|<|Y|，且|Y|!=0

[X]原=1.10001011，[Y]原=0.1110，[-Y]补=1.0010

商的符号=1 XOR 0=1

恢复余数法做除法最大的缺点是，不同的被除数和除数，其除的过程不规范，何时需要恢复余数是不相同的，实现时不便于控制。因此，恢复余数法在计算机中不常用，但由此法可演变出另一种更方便的计算方法，这就是加减交替法。

加减交替法

①其过程是符号位单独处理：商符号为被除数和除数符号异或的结果，余数的符号和被除数相同

②初始余数（被除数）左移1位，减除数（-|Y|）

③若余数 R≥0，则商上 1，左移1次，减除数

④若余数 R＜0，则商上 0，左移1次，加除数

重复上述的步骤③④直到除尽或精度达到要求。最后， 拼接商符得到商，即可获得除法的结果。余数的尾数为余数（被除数）列中在不断左移的过程中仍保留的数（不理解可以看下面的过程，加粗字为仍保留的数，细体为左移新生成的数），余数的阶数为-1*[左移次数]

下面举例说明加减交替法做除法的过程：

若被除数数 X＝－0.10001011，除数 Y＝0.1110，利用原码求商及余数。

该例满足|X|<|Y|，且|Y|!=0

[X]原=1.10001011，[Y]原=0.1110，[-Y]补=1.0010

商的符号=1 XOR 0=1

绝对值除法中-|Y|是利用补码加法实现的，即+[-Y]补，过程如下

补码除法法则

同样，假定进行定点纯小数的补码除法运算，其先决条件是除数≠0 且|被除数|＜|除数|。注意：需要带上符号一起计算。

① 首先，如果被除数与除数同号，被除数减除数；如果被除数与除数异号，被除数加除数。计算的结果均称为余数。

② 若余数与除数同号，上商为1，余数左移1次，下次用余数减除数操作求商；若余数与除数异号，上商为0，余数左移1次， 下次用余数加除数操作求商。

③ 重复②直至除尽或达到进度要求

④ 商修正。在除不尽时，通常可用商的最低位恒置1进行修正来保证精度。

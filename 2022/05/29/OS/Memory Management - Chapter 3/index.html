Memory Management - Chapter 3

内存的基础知识

CPU Utilization

If there are n programs running in the memory, the CPU utilization is $1-P^n$

P：等待IO的时间的占比，$0\leq P<1$

由该式易得：等待IO的时间占比越小，在CPU中运行的程序越多，CPU利用率越高

而提升内存的大小可以增加内存中运行的程序数量，从而提高CPU利用率

Relocation

将图中指令中的逻辑地址信息转换为对应的物理地址的方式叫做Relocation.

绝对装入

在编译时，如果知道程序将放到内存中的哪个位置，编译程序将产生绝对地址的目标代码。装入程序按照装入模块中的地址，将程序和数据装入内存。

静态重定位

又称可重定位装入。

在装入时对地址进行重定位，将逻辑地址变为物理地址。

在一个作业装入内存的时候，必须分配其要求的全部内存空间，如果没有足够的内存，就不能装入该作业。作业一旦装入，在运行期间就不能再移动，也不能再申请内存空间。

动态重定位

又称动态运行时装入。

把地址转换推迟到程序真正要运行时才进行。

动态重定位利用重定位寄存器（存放装入模块存放的起始位置）进行地址转换。

采用动态重定位时允许程序在内存中发生移动。

Link

静态链接

在程序运行之前，先将各目标模块及他们所需的库函数连成一个完整的可执行文件（装入模块），之后不在拆开。

装入时动态链接

将各目标模块装入内存时，边装入边链接的链接方式。

运行时动态链接

在程序执行中需要该目标模块时，才对它进行链接。其优点是便于修改和更新，便于实现对目标模块的共享。

存储保护

设置上下限寄存器

在CPU中设置一对上、下限寄存器，存放进程的上下限物理地址。

Base and Limit

Base & Limit 都是作为寄存器储存在PCB中, 每个进程都有一个.

Base：进程基地址

Limit：进程拥有的最大地址数

Base + Limit对应进程在物理地址中的最大地址

用于给进程限定内存空间，起到内存保护的作用。

Swapping

设计思想

内存空间紧张时，系统将内存中某些进程暂时换出到外存，把外存中某些已具备运行条件的进程换入到内存。

即进程在内存与磁盘间动态调度。

Memory allocation changes when

  • process leave memory

Memory Management

Bitmap

Bitmap是用于管理内存的一种数据结构。

  • 在Bitmap中，Memory被分割成固定等大的allocation unit，Bitmap中的每一位都对应着一个allocation unit

  • 0表示该unit空闲，1表示该unit已被占用

The size of the bitmap depends on:

  • the size of memory
  • the size of allocation unit

优点：修改方便，使用取非操作即可

缺点：在决定把一个占k个allocation unit的进程调入内存时，存储管理器必须搜索Bitmap，在Bitmap中找到有k个连续的0的串，而查找Bitmap中指定长度的连续0串是很耗时的。

Linked List

使用链表管理内存。

  • 每个Node代表一块空闲或被占用的内存

  • 每个Node包含：

    • 指示标志：H（空闲）或P（进程占用）
    • 起始地址
    • 内存块长度（size）
    • 指向下一内存块对应的Node的指针域
  • 采用双向链表，便于和前后节点合并
  • 在书上例子中，链表Node按地址排序，也可按其它方式排序

优点：便于查找连续的空闲内存块

缺点：修改麻烦，修改后需要考虑与前后Node合并等问题

Algorithm for Dynamic Memory Allocation

为创建的进程寻找合适的hole的算法。

First fit

从0开始找到第一个合适的hole

Next fit

从上一次查找结束的地方开始找到第一个合适的hole

Best fit

在所有hole中寻找满足进程的Memory需求并且size最小的hole

缺点：查找hole的速度慢，并且会产生大量无法利用的小空闲区，反而浪费了更多内存

Worst fit

在所有hole中寻找满足进程的Memory需求并且size最大的hole

缺点：查找hole的速度慢

Quick fit

随机查找hole，只要能满足进程的需求就占用这个holePage

Quick fit是这些算法中实际性能最好的算法

Virtual Memory

定义：程序不需要全部装入内存即可运行，运行时根据需要动态调入数据，若内存不够则从内存换出一些数据到外存。

和Swapping的区别是，Swapping是整个进程的调入调出，而Virtual Memory是进程的部分的调入调出。

MMU (Memory Management Unit)

MMU maps the virtual addresses onto the physical memory addresses.

Paging

Page

按照固定大小划分虚拟地址空间得到的若干个单元被称为页面（Page）

Page Frame

Page在物理内存中对应的单元称为Page Frame

Page Fault

When MMU notices that the page is unmapped and causes the CPU to trap to the operating system. This trap is called a page fault.

Page Table

  • 页表（page table）用于记录进程页面和实际存放的内存块（页框）之间的对应关系。

  • 一个进程对应一张页表，进程的每一页对应一个页表项，每个页表项由“页号”和“块号”组成。

  • 每个页表项的长度是相同的，页号隐含在页表项的顺序中。

  • 页表项的长度取决于页框数，如：

    • 问：某系统物理内存大小为4GB，页面大小为4KB，则每个页表项至少应为多少字节？
    • 答：有4GB/4KB = $2^{20}$ 个页框，因此需要20个bit表示$2^{20}$ 个页框，至少3个字节

Typical Page Table Entry

  • Page frame number 对应的页框号
  • Present /absent 若为1 ,表示该页对应物理地址在内存中
  • Protection 标识访问权限, R/W/X
  • Modified 标识是否修改过, 若1, 调出时需要写入磁盘
  • Referenced 最近是否被访问, 用于页面调度算法的参考
  • Caching disabled 用于禁止该页进入cache

页号、页内偏移量的计算

对于十进制的地址来说：

  • 页号=逻辑地址/页面大小 （向下取整）
  • 页内偏移量=逻辑地址%页面大小

然而在计算机中，一个进程对应的页号和页内偏移量是放在一个二进制串中的：

  • 高位表示页号

  • 低位表示页内偏移量

  • 如：假设每个页面的大小为4KB，页表中共20个页表项，某个逻辑地址在2号页的进程可能的逻辑地址为00000000000000000010000000000000 ~ 00000000000000000010111111111111

    其中前20位表示页号，后12位表示偏移量

地址转换

  1. 计算出逻辑地址高位对应的页号p
  2. 在页表中[0…n]找到[p]号页表项对应的页框f
  3. 页框f和偏移量d组合得到物理地址

Translation Lookaside Buffer (TLB)

TLB用于解决页表存储在内存中，而页表访问次数又很多的情况下时间消耗较大的问题。

根据局部性原理，将最近常访问的少量页面记录在快表（TLB）中，快表通常在MMU中。

工作过程：

  1. 在想要查询某页对应的页框时，先查询TLB中是否存储相应信息
  2. 若TLB中存储了该页表项信息，则可以不用访问内存而是从TLB中获取
  3. 若TLB中未存储该页表项信息，则需要访问内存获取，并且在访问内存获取到对应的页框后将信息更新到TLB中以便下次访问

软失效（soft miss）：当一个页面在内存中而不在TLB中时，将产生软失效

硬失效（hard miss）：当一个页面不在TLB中且不在内存中时，将产生硬失效

Multilevel Page Table

多级页表（Multilevel Page Table）用于解决在内存较大的情况下，页表本身所占的内存过大且只能使用连续的内存空间存储页表的问题。

Page Replacement Algorithm

Optimal Page Replacement Algorithm

每次都将未来最远被用到的页面调出既是最优的页面置换算法。

然而这种算法难以实现，人们无法预测未来页面的调用请求。

The best possible page replacement algorithm is easy to describe but impossible to actually implement.

The Not Recently Used Page Replacement Algorithm

每个页都有 Reference bit， Modified bit，根据两个位的信息对页未来被调用的远近进行评估，按照以下顺序调出：

  1. not referenced, not modified.
  2. not referenced, modified.
  3. referenced, not modified.
  4. referenced, modified.

The First-In, First-Out (FIFO) Page Replacement Algorithm

建立一个在内存中的页队列，每调入一个页就加入到队尾，每次需要调出页的时候就将队首的页调出内存，将要调入的页加入到队尾，从而实现了“先进先出”。

这种算法在内存增大的情况下，缺页中断的次数反而发生的更多，其问题在于这种算法有可能将经常访问修改的页反复调入调出。

The Second-Chance Page Replacement Algorithm

该算法是基于FIFO算法的改进。

利用ReferenceBit可以判断某页是否最近被访问，从而避免常访问的页被频繁调入调出。

如果在队列中找到某页面，先检查其ReferenceBit：

  • 若ReferenceBit为0，则说明该页最近未被访问，那么我们调出该页。
  • 若ReferenceBit为1，则说明该页最近被访问过，那么我们给他一次机会，将其ReferenceBit置零，这样的话，下次需要置换页面的时候如果该页面在我们给了机会后又被访问了就其ReferenceBit就还是1，避免了被频繁调出。

The Clock Page Replacement Algorithm

和The Second-Chance PRA思想相同，只是使用了循环队列实现，由于循环队列遍历形似一个指针不停转动的时钟，因此叫The Clock PRA。

如果遍历到队列中的某页：

  • 若ReferenceBit为0，则说明该页最近未被访问，那么我们调出该页。
  • 若ReferenceBit为1，则说明该页最近被访问过，那么我们将其ReferenceBit置零，指针后移。

The Least Recently Used (LRU) Page Replacement Algorithm

一种较为接近最优页面置换算法的可实现算法，其算法思想是调出最近最少使用的页。

Simulating LRU in Software

使用矩阵模拟

内存中有n个页时，建立一个n*n的矩阵

  • 当访问第k页时，第k列的全部值置0，第k行的其他值全部置1
  • 当需要调出时，调出行累加和最少的
  • 此方式开销较大

The Aging Algorithm Simulating LRU

  1. 对于内存中的每个页，维护一个多位二进制标识值，初始每一位都为0

  2. 每次时钟滴答，根据内存中的页的ReferenceBit可以得到他们此周期是否被访问，将每一页的标识值的最高位赋值为该页对应的ReferenceBit，然后右移一位

  3. 需要页面置换时，选择标识值最小的页调出

The Working Set Page Replacement Algorithm

基于历史访问记录的预测来确定一个工作集（Working Set），当需要置换的时候置换掉不在工作集中的页面。

工作集：最近k次内存访问过的页面集合，$w(k,t)$是在t时刻工作集的大小

然而理想的工作集页面置换算法实现起来开销较大，可使用以下近似方法。

并不储存最近k次内存访问的页面，而是储存最近一段时间内访问的页面

算法流程

如果没有R==0且生存时间大于设定的t的页面时，就调出R==0且生存时间最长（上次使用时间最小）的页面；

考虑最坏的情况，如果没有R==0的页面，那么我们随机调出一个页面。

The WSClock Page Replacement Algorithm

就是工作集页面置换算法+The Clock PRA，这里不再赘述。

Summary of Page Replacement Algorithms

Algorithm Comment
Optimal Not implementable, but useful as a benchmark
NRU (Not Recently Used) Very crude approximation of LRU
FIFO (First-In, First-Out) Might throw out important pages
Second chance Big improvement over FIFO
Clock Realistic
LRU (Least Recently Used) Excellent, but difficult to implement exactly
NFU (Not Frequently Used) Fairly crude approximation to LRU
Aging Efficient algor ithm that approximates LRU well
Working set Somewhat expensive to implement
WSClock Good efficient algorithm

Design Issues for Paging System

此小节非重点，这部分主要参考了学长的博客（准确地说是大段地搬运）

Page Fault Handling

  1. trap 进内核, 保存PC
  2. 保存进程寄存器
  3. OS 确定调入页
  4. OS 确定调出页
  5. 若调出页修改过(dirty), 需要写入磁盘
  6. 调入调入页
  7. 更新页表
  8. 恢复到页面失效的指令
  9. 恢复进程
  10. 恢复寄存器, 恢复PC

Local versus Global Allocation Policies

多进程并发时,内存中所有的页统一调度 VS 只在该进程拥有的页内部调度

全局调度页面会使得一段时间后内存的利用趋近于实际进程所需的比例.

Load Control

当 PFF (page fault frequency 页面失效频率) 过高时, 主动减少使用内存的进程.

Page Size

Page size应当较小，typically 4KB or 8KB

Advantage：

  • less internal fragmentation
  • better fit for various data structures and code sections
  • less unused program in memory

Disadvantage：

  • programs need many pages, larger page tables

Separate Instruction and Data Spaces

Shared Pages

程序的代码段可以共享，但数据段一般不共享，其情况较为复杂。

Cleaning Policy

为了保证在新进程创建的时候有足够的空白页面支持， 必须时刻保证内存中有足够的空白页。

paging daemon （分页守护进程）：当空白页面不足时，会使用页面置换算法主动调出页，但不调入页帧。

也可使用clock

Operating System Involvement with Paging

Four times when OS involved with paging

  1. Process creation
  2. Process execution
  3. Page fault time
  4. Process termination time

Segmentation

根据程序储存的信息的类型分段储存，程序的数据包括代码段(Code), 数据段(Data), 堆栈段(Stack)等等。

每一段的逻辑地址都从0 开始，每一段都拥有自己的Base 与 Limit ，这些映射关系储存在段表中。

Advantages

  • 简化正在增长或缩小的数据结构的处理

  • 简化了分别编译的过程的链接

  • 便于在多个进程之间共享过程或数据
  • 不同的段可以具有不同种类的保护策略

LDT & GDT

Local descriptor table(LDT)是每个程序都拥有的一张段表，存储着不同段的base和limit，并且如页表一样，将段号隐存在储存顺序中。

Global descriptor table(GDT)是一台计算机上所有程序共享的一张表，用于描述系统段，包括操作系统本身。

Selector

选择子（selector）储存在对应的段寄存器中（如代码段选择子储存在代码段寄存器中），用于存储段号和GDT/LDT属性和privilege level，
